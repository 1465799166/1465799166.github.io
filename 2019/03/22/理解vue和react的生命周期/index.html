分析

  1. 从new Vue()开始，一个vue组件便被创建了，此时会开始初始化Vue对象的属性和方法，这和new任何一个对象都是一样的
  2. 初始化完成后，vue首先会去获取props属性下的值（图中的第一行undefined），之后会将其合并到data中，并进行监听
  3. 进入beforeCreate，此时el仍未初始化，data方法已声明，但data仍未初始化
  4. 进入created，此时el仍未初始化，data已完成初始化，并实现监听
  5. 判断是否声明el，若声明则进入下一步，若未声明，则等待手动调用vue执行vm.$mount()方法，再进入下一步
  6. 判断是否声明template，如果有，则使用template作为vm（viewmodel）的模板对象，如果没有，则使用外部html（即挂载目标#app内的html部分）作为vm的模板对象
  7. 进入beforeMount，此时el已初始化，但vm仍以virtual-dom的形式占位，并在该周期开始进行dom的render
  8. 进入mount，此时virtual-dom被替换为真实的dom
  9. 当data监听的属性发生变化时，触发beforeupdate钩子，使用diff算法比较差异，重新render，并进入updated钩子
  10. 当vm.destory()触发时，进入beforeDestroy钩子，卸载当前vm及其子组件，并进入destroyed钩子

分析

初始化

  • defaultProps & propTypes 为props设置默认值及type ，同vue props:{type: & default: }
  • constructor {super & state} 构造函数，继承父组件props，初始化state
  • componentWillMount 组件挂载前，只执行一次，一般在这里进行http请求 相当于vue beforeMount
  • render组件渲染
  • componentDidMount 组件挂载完成 此时dom真实存在，可进行相应dom操作，在此处改变state已经会进入shouldComponentUpdate钩子,但是在当前钩子仍然无法获取改变后的state

    运行时

  • componentWillReceiveProps 父组件Props发生改变时触发，注意：初始化获取props并不会触发这个钩子
  • shouldComponentUpdate 当父组件props改变或当前组件state改变时， 决定当前组件是否需要重新渲染，return true 代表会，即重新执行render钩子,触发update
  • componentWillUpdate 当shouldComponentUpdate返回true时，进入该钩子，然后进行重新渲染（render）
  • componentDidUpdate 此时组件更新完成

销毁阶段

  • componentWillUnmount 组件销毁
